## Before Contributing

Before proposing any code, documentation, design, or deployment change, review:

- `LICENSE.md`
- `NOTICE.md`
- `SECURITY.md`
- `SETUP.md`

## Contribution Policy

This repository is not open for unrestricted public contribution.

Contributions should be made only when:

- you are explicitly authorized by the owner
- your work is within an agreed project scope
- you understand that the repository is not free to use

Submitting a patch, suggestion, branch, or documentation update does not grant any ownership transfer, reuse license, redistribution right, or independent product right.

## Security-Related Contributions

If your contribution addresses a vulnerability:

- do not open a public issue with exploit details
- report privately using the process in `SECURITY.md`
- avoid including secrets, tokens, or sensitive production data in examples

## Code and Documentation Expectations

- keep environment secrets out of version control
- preserve ownership and legal notices
- document meaningful security-impacting changes
- do not introduce third-party assets or code with incompatible licensing
- keep setup instructions accurate when behavior changes

## Review and Acceptance

All contributions are reviewed at the owner's discretion. The owner may accept, modify, defer, or reject contributions for technical, legal, security, operational, or product reasons.
